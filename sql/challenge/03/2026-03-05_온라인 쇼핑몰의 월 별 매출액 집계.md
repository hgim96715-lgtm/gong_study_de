##  해결 전략 (Code Before Think)

> `order_id`가 C로 시작하는 주문이 취소 주문입니다. 결과 데이터는 아래 4개 컬럼을 포함해야 하고 `order_month` 컬럼의 값으로 오름차순 정렬되어 있어야 합니다.
> - `order_month`: YYYY-MM 형식으로 표기된 주문 연, 월 정보 `ordered_amount`: 취소 주문을 제외한 주문 금액의 합계 `canceled_amount`: 취소 주문의 금액 합계 (음수로 표시) `total_amount`: 취소 주문을 포함한 주문 금액의 총 합계

1. **타겟 데이터:** (어떤 테이블에서 무엇을 뽑아야 하는가?)
   - `orders`  테이블
	   - `order_id` 거래 ID (C로 시작하는 경우 환불)
	   - `order_date` 거래일
   - `order_items` 테이블
	   - `order_id` 거래 ID (C로 시작하는 경우 환불)
	   - `price` 1개 당 가격 (파운드)
	   - `quantity` 수량 (음수인 경우 환불)
1. **조건 분석:**
   - `order_id`가 **`C`로 시작하는 주문은 취소 주문**으로 분류
   - 월 별 집계를 위해 주문일(`order_date`)을 **`YYYY-MM` 형식**으로 변환
1. **사용할 문법:**
   - `TO_CHAR(date, 'YYYY-MM')`
   - `FILTER (WHERE ...)`

## 정답 쿼리 (Solution)

```sql
-- 여기에 작성한 쿼리를 붙여넣으세요.
SELECT TO_CHAR(o.order_date,'YYYY-MM') as order_month,
SUM(oi.price*oi.quantity) FILTER (WHERE oi.order_id NOT ILIKE 'C%') as ordered_amount,
SUM(oi.price*oi.quantity) FILTER (WHERE oi.order_id ILIKE 'C%') as canceled_amount,
SUM(oi.price*oi.quantity) as total_amount
FROM order_items oi
JOIN orders o ON oi.order_id=o.order_id
GROUP BY TO_CHAR(o.order_date,'YYYY-MM')
ORDER BY order_month
```
