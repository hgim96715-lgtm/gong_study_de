## 정답 쿼리 (Solution)

```sql
-- 여기에 작성한 쿼리를 붙여넣으세요.
SELECT
	e1.employee_id AS mentee_id,
	e1.name AS mentee_name,
	e2.employee_id AS mentor_id,
	e2.name AS mentor_name
FROM employees e1
LEFT JOIN employees e2
	ON e1.department != e2.department
	AND e1.employee_id != e2.employee_id
	AND e2.join_date <= ('20211231'::DATE - INTERVAL '2 year')::DATE
	WHERE e1.join_date >= ('20211231'::DATE - INTERVAL '3 month')::DATE
ORDER BY
	mentee_id ASC,
	mentor_id ASC;
```

##  오답 노트 & 배운 점 (Retrospective)

-  **내가 실수한 부분:**
	- 멘토 조건(`재직 2년 이상`)을 **ON 절이 아니라 WHERE 절에 작성하려고 했음** >이로 인해 `LEFT JOIN`이 **INNER JOIN처럼 동작**할 수 있다는 점을 놓침
	- 멘티와 멘토를 한 테이블에서 구분하려고 `CASE WHEN`으로 역할을 나누는 방식으로 접근함 >결과적으로 **JOIN 조건과 역할 분리가 복잡해짐**
-  **새로 알게 된 함수/꿀팁:**
	- 멘토링 문제는 CASE 문제가 아니라 JOIN 설계 문제
	- 조인 대상 테이블(있으면 붙고, 없으면 NULL)의 조건 → `ON`
	- 기준 테이블(반드시 나와야 하는 집합)의 조건 → `WHERE`

>“없어도 나와야 한다”는 조건은 CASE가 아니라 LEFT JOIN + ON 절 위치로 해결한다.
