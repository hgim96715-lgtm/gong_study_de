##  해결 전략 (Code Before Think)

> 2017년 1월 한 달 동안 발생한 주문의 배송 예측이 정확했는지 분석을 하려고 합니다. 고객의 구매 일자별로 배송 예정 시각 안에 고객에게 도착한 주문과, 배송 예정 시각이 지나서 고객에게 도착한 주문을 각각 집계하는 쿼리를 작성해주세요. 배송 완료 또는 배송 예정 시각 데이터가 없는 경우는 계산에서 제외합니다. 계산 결과는 구매 날짜를 기준으로 오름차순 정렬되어야 한다.

1. **타겟 데이터:** (어떤 테이블에서 무엇을 뽑아야 하는가?)
   - `olist_orders_dataset` 테이블
	   - `order_id` : 주문 식별
	   - `order_purchase_timestamp` : 구매 시각
	   - `order_estimated_delivery_date` : 배송 예정 시각
	   -  `order_delivered_customer_date` : 실제 배송 완료 시각
1. **조건 분석:**
   - `order_purchase_timestamp`를 **DATE로 변환하여 GROUP BY**
1. **사용할 문법:**
   - `EXTRACT()` : 연도와 월 필터링
   - `FILTER()` : 조건별 집계
---
## 정답 쿼리 (Solution)

```sql
-- 여기에 작성한 쿼리를 붙여넣으세요.
SELECT order_purchase_timestamp::DATE as purchase_date,
COUNT(DISTINCT order_id) FILTER (
    WHERE order_estimated_delivery_date::DATE - order_delivered_customer_date::DATE > 0
) as success,
COUNT(DISTINCT order_id) FILTER (
    WHERE order_estimated_delivery_date::DATE - order_delivered_customer_date::DATE <= 0
) as fail
FROM olist_orders_dataset
WHERE EXTRACT(YEAR FROM order_purchase_timestamp::DATE) = 2017
AND EXTRACT(MONTH FROM order_purchase_timestamp::DATE) = 1
GROUP BY order_purchase_timestamp::DATE
ORDER BY purchase_date
```

## 더 나은 풀이가 있다면?

```sql
-- 더 나은 풀이가 있을경우 
SELECT 
    order_purchase_timestamp::DATE AS purchase_date,
    COUNT(DISTINCT order_id) FILTER (
        WHERE order_delivered_customer_date <= order_estimated_delivery_date
    ) AS success,
    COUNT(DISTINCT order_id) FILTER (
        WHERE order_delivered_customer_date > order_estimated_delivery_date
    ) AS fail
FROM olist_orders_dataset
WHERE order_purchase_timestamp >= '2017-01-01'
AND order_purchase_timestamp < '2017-02-01'
AND order_delivered_customer_date IS NOT NULL
AND order_estimated_delivery_date IS NOT NULL
GROUP BY purchase_date
ORDER BY purchase_date;
```
