##  해결 전략 (Code Before Think)

> 2019년 10월 한 달 동안 정류소에서 발생한 대여/반납 건수가 2018년 10월 같은 정류소에서 발생한 대여/반납 건수의 50% 이하인 정류소를 출력하는 쿼리를 작성해주세요. 2018년 10월 또는 2019년 10월 한 달간 대여/반납 건수가 0건인 정류소는 이미 폐쇄된 정류소이거나 새로이 생긴 정류소 일 수 있으므로 쿼리 결과에 포함되지 않도록 해주세요

1. **타겟 데이터:** (어떤 테이블에서 무엇을 뽑아야 하는가?)

## 정답 쿼리 (Solution)

```sql
-- 여기에 작성한 쿼리를 붙여넣으세요.
WITH sum_pct AS (
    SELECT
        s.station_id,
        s.name,
        s.local,
        SUM(1) FILTER (
            WHERE (rh.rent_at >= '2018-10-01' AND rh.rent_at < '2018-11-01') 
               OR (rh.return_at >= '2018-10-01' AND rh.return_at < '2018-11-01')
        ) AS sum_2018,
        SUM(1) FILTER (
            WHERE (rh.rent_at >= '2019-10-01' AND rh.rent_at < '2019-11-01') 
               OR (rh.return_at >= '2019-10-01' AND rh.return_at < '2019-11-01')
        ) AS sum_2019
    FROM station s
    LEFT JOIN (
        SELECT rent_station_id AS station_id, rent_at, return_at
        FROM rental_history
        UNION ALL
        SELECT return_station_id AS station_id, rent_at, return_at
        FROM rental_history
    ) rh ON s.station_id = rh.station_id
    GROUP BY s.station_id, s.name, s.local
)
SELECT
    station_id,
    name,
    local,
    ROUND(sum_2019::numeric / sum_2018 * 100, 2) AS usage_pct
FROM sum_pct
WHERE sum_2018 > 0
  AND sum_2019 > 0
  AND sum_2019 <= sum_2018 * 0.5
ORDER BY station_id;
```

##  오답 노트 & 배운 점 (Retrospective)

-  **내가 실수한 부분:**
	- `LEFT JOIN ... OR ...` 조건으로 조인을 시도 
	- 왜 그런지 찾아보니 → PostgreSQL에서 **성능 저하** 로 인해 실행이 안되었다. 
-  **새로 알게 된 함수/꿀팁:**
	- **UNION ALL을 사용하여 rent_station_id와 return_station_id를 합쳐서 조인** → OR 조건 없이도 같은 결과 얻기 가능
